# Contributing to Smart Lighting Control System
# المساهمة في نظام التحكم الذكي بالإضاءة

شكراً لاهتمامك بالمساهمة في هذا المشروع! 🎉  
Thank you for your interest in contributing to this project! 🎉

## 🌟 كيفية المساهمة | How to Contribute

### 1. Fork المشروع
انقر على زر "Fork" في أعلى الصفحة

### 2. Clone المستودع
```bash
git clone https://github.com/your-username/smart-lighting-system.git
cd smart-lighting-system
```

### 3. أنشئ فرع جديد
```bash
git checkout -b feature/your-feature-name
```

### 4. قم بالتعديلات
- اكتب كود نظيف ومُعلق
- اتبع نمط الكتابة الموجود
- اختبر التغييرات جيداً

### 5. Commit التغييرات
```bash
git add .
git commit -m "Add: وصف واضح للتغيير"
```

### 6. Push إلى GitHub
```bash
git push origin feature/your-feature-name
```

### 7. افتح Pull Request
اذهب إلى صفحة المشروع الأصلي وانقر "New Pull Request"

## 📝 إرشادات الكتابة | Coding Guidelines

### Arduino/C++
- استخدم أسماء متغيرات واضحة ومفهومة
- أضف تعليقات بالعربية والإنجليزية
- اتبع معايير Arduino Code Style

### التعليقات | Comments
```cpp
// ✅ جيد | Good
int sensorValue = 0;  // قراءة المستشعر | Sensor reading

// ❌ سيء | Bad
int x = 0;  // قيمة
```

### التسميات | Naming
- المتغيرات: `camelCase`
- الثوابت: `UPPER_CASE`
- الدوال: `camelCase`

## 🐛 الإبلاغ عن الأخطاء | Reporting Bugs

عند الإبلاغ عن خطأ، يُرجى تضمين:

When reporting a bug, please include:

1. **وصف المشكلة** | Problem Description
2. **خطوات إعادة الإنتاج** | Steps to Reproduce
3. **السلوك المتوقع** | Expected Behavior
4. **السلوك الفعلي** | Actual Behavior
5. **البيئة** | Environment:
   - نوع اللوحة (ESP8266/ESP32)
   - إصدار Arduino IDE
   - إصدار المكتبات

---

## 💡 اقتراح ميزات | Suggesting Features

نرحب بالأفكار الجديدة! افتح Issue بـ:

We welcome new ideas! Open an issue with:

- **وصف الميزة** | Feature Description
- **الفائدة المتوقعة** | Expected Benefit
- **أمثلة الاستخدام** | Use Cases
- **بدائل محتملة** | Possible Alternatives

---

## ✅ قائمة التحقق | Checklist

قبل إرسال Pull Request:

Before submitting a Pull Request:

- [ ] تم اختبار الكود على الأجهزة الفعلية
- [ ] تم تحديث التوثيق (إذا لزم الأمر)
- [ ] تم إضافة تعليقات واضحة
- [ ] لا توجد معلومات حساسة في الكود
- [ ] الكود يتبع نمط المشروع
- [ ] تم تحديث CHANGELOG.md
